windows11实现docker和vm共存

发表于2026-04-07|更新于2026-04-22
|浏览量:

是宿主机中的docker、WSL2和VM虚拟机共存

打开windows的虚拟化平台

如图所示打开windows的虚拟化平台，重启电脑后生效

关闭虚拟机的虚拟化引擎

VMware15.5版本以上已经支持虚拟化平台和虚拟机共存

影响：无法在虚拟机内再嵌套虚拟化（比如在虚拟机里开 Docker、WSL2），日常使用完全无影响

关闭后打开会显示

关闭侧通道缓解

  • 原理：侧通道缓解是为了防御 Spectre/Meltdown 等 CPU 侧信道攻击的安全机制，开启后会显著降低虚拟机性能（卡顿、响应慢）
  • 触发原因：你的物理机 Windows 10 开启了 Hyper-V / 内核隔离等虚拟化安全功能，VMware 自动启用了该缓解措施

彻底关闭侧通道缓解（提升性能，推荐）

前置要求：必须先完全关闭虚拟机（不能挂起）

  1. 选中该 Windows 10 虚拟机，点击菜单栏 「虚拟机 (VM)」→「设置 (Settings)」
  2. 在弹出的窗口中，切换到 「选项 (Options)」 标签页找到左侧 「高级 (Advanced)」 选项
  3. 在右侧勾选 「不为启用了Hyper-V的主机启用侧通道缓解」
  4. 点击「确定」保存，重新启动虚拟机即可

![屏幕截图 2026-04-07 121738](../img/屏幕截图 2026-04-07 121738.png)

选项 安全性 性能 适用场景
开启侧通道缓解 高 低 处理敏感数据、高安全需求的生产环境
关闭侧通道缓解 降低 高 个人日常使用、开发测试、追求流畅体验
